# 2. Vytvoření základní politiky

Při zakládání **Agent Policy** pro komponentu **API-Gateway** postupujte v rozhraní Kibana následovně. Tento postup zajistí, že politika bude připravena pro všechna tři prostředí (DEV, TEST, PROD) a bude správně sbírat data z vašich souborů.

---

## 1. Vytvoření základní politiky

1. V levém menu přejděte na **Management** -> **Fleet**.
2. Klikněte na záložku **Agent Policies** a poté na tlačítko **Create agent policy**.
3. Vyplňte pole:
    - **Name:** `API-Gateway-Policy`
    - **Description:** Např. *„Společná politika pro API Gateway servery napříč prostředími“*.
    - **Namespace:** Zde nechte pro tuto chvíli `default`.

4. > **Poznámka:** Konkrétní prostředí jako `dev` nebo `prod` budeme definovat až u samotné integrace nebo při instalaci agenta, aby tato jedna politika byla znovupoužitelná.

5. Ponechte zaškrtnuté **Collect system logs and metrics** (budete mít přehled o CPU/RAM serveru, kde API běží).
6. Klikněte na **Create agent policy**.

---

## 2. Přidání sběru logů (Integrace Custom Logs)

Nyní musíte politice říct, kde přesně má logy API Gateway hledat:

1. V seznamu politik klikněte na nově vytvořenou **API-Gateway-Policy**.
2. Klikněte na **Add integration**.
3. Vyhledejte a vyberte **Custom Logs**.
4. Klikněte na **Add Custom Logs** a vyplňte:
    - **Integration name:** `api-gw-logs-collection`
    - **Log file path:** Zadejte absolutní cestu k logům na serverech.
        - *Tip:* Pokud se cesta na DEV/PROD liší, můžete použít zástupné znaky (např. `C:\App\Logs\*.log` nebo `/var/log/api-gw/*.log`).
    - **Dataset name:** Zadejte `api.gw`. (To je část názvu, která tvoří `logs-api.gw-dev`).
    - **Namespace:** Nechte `default`.

5. **Nastavení kódování (Důležité):**
    Protože vaše zdrojové logy jsou v kódování **windows-1250** (středoevropské jazyky), musíte toto nastavit v **Advanced options**:
    - Rozbalte **Advanced options**.
    - V poli **Encoding** vyberte nebo napište `windows-1250`.
    - *(Pokud kódování nenastavíte, budou se české/slovenské znaky v Elasticu zobrazovat chybně jako čtverečky nebo otazníky).*

6. > **Důležité:** Skutečný namespace (`dev`/`test`/`prod`) určíte až v momentě, kdy budete agenta fyzicky instalovat na daný server (pomocí parametru `--namespace`).

---

## 3. Nastavení pro Correlation ID (Parsace logů)

Aby bylo možné hledat podle `correlationId`, musí Elastic vědět, kde se v textu logu nachází.

1. V nastavení téže integrace (**Custom Logs**) sjeďte dolů na **Custom configurations**.
2. Zde můžete definovat **Ingest Pipeline** (pokud ji máte připravenou) nebo v sekci **Processors** přidat jednoduchý regulární výraz pro vytažení ID.

3. > **Tip:** Pokud jsou vaše logy ve formátu **JSON**, stačí zaškrtnout volbu **Parse JSON** a Elastic pole `correlationId` vytvoří automaticky.

---

## 4. Uložení

1. Klikněte na **Save and continue**.
2. Potvrďte uložení politiky tlačítkem **Save and deploy changes**.

---

### Výsledek
Máte připravenou „šablonu“ pro všechny API Gateway servery. Až budete instalovat agenta na DEV server, použijete při instalaci tuto politiku a k příkazu přidáte `--namespace dev`. Tím se data automaticky začnou sypat do `logs-api.gw-dev`.

---

**Další kroky:**
1. Chcete nyní pomoci s nastavením regulárního výrazu (**Grok**) pro vytažení vašeho `correlationId`, pokud vaše logy nejsou v JSON formátu?
