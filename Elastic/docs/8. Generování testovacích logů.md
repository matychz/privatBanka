# 8. Generování testovacích logů

Pro účely testování a simulace reálného provozu byl vytvořen generátor logů, který do souboru `Elastic/logs/api-gw/privat.log` připisuje nové řádky ve správném formátu a kódování (`windows-1250`).

---

## 1. Umístění skriptů

Skripty se nacházejí v adresáři `Elastic/scripts/`:

- `log_generator.py`: Hlavní logika generátoru v Pythonu.
- `run_generator.ps1`: PowerShell skript pro snadné spuštění ve Windows 11.
- `run_generator.bat`: Dávkový soubor pro spuštění dvojklikem.

---

## 2. Jak spustit generátor

### Možnost A: Přes PowerShell (Windows 11)
1. Otevřete terminál v kořenovém adresáři projektu.
2. Spusťte skript:
   ```powershell
   .\Elastic\scripts\run_generator.ps1
   ```

### Možnost B: Dvojklikem (Průzkumník)
1. Přejděte do složky `Elastic\scripts`.
2. Spusťte soubor `run_generator.bat`.

---

## 3. Chování generátoru

- **Průběžný zápis:** Skript běží v nekonečné smyčce a každých 0.5 až 2 sekundy zapíše jeden nový řádek.
- **Levely:** Skript generuje `INFO`, `DEBUG`, `WARN` a `ERROR` levely, což umožňuje testování dashboardů, vizualizací chybovosti a alertingu.
- **Formát:** Generuje realistické zprávy pro komponentu API Gateway (Health checky, Onboarding procesy, odesílání e-mailů).
- **ID:** Každý řádek obsahuje náhodné `traceId` nebo `processId`.
- **Kódování:** Text je kódován jako `windows-1250` (pro správnou českou diakritiku v Elasticu).
- **Ukončení:** Generování zastavíte stisknutím **Ctrl+C** v okně terminálu.

---

## 4. Konfigurace (volitelné)

Pokud chcete změnit frekvenci generování, můžete upravit proměnnou `DELAY_SECONDS` v souboru `log_generator.py`:

```python
# log_generator.py
DELAY_SECONDS = (0.1, 0.5)  # Rychlejší generování (v sekundách)
```

---

> **Tip:** Generátor je ideální pro testování Ingest Pipeline v reálném čase, kdy můžete sledovat, jak se nově přidané logy okamžitě objevují v Kibaně prostřednictvím Elastic Agenta.
