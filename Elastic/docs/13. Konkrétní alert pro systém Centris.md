# 13. Konkrétní alert pro systém Centris a jeho import v JSON formátu

Tento dokument obsahuje konfiguraci alertu pro detekci chyby spojení se systémem Centris a popisuje, jak lze alerty v Kibaně hromadně spravovat pomocí formátu **NDJSON** (Saved Objects).

---

## 1. Konfigurace alertu (Logika)

Pro detekci chyby spojení se systémem Centris nastavte v Kibaně pravidlo (Rule) s těmito parametry:

- **Název:** `System Centris - Connection Failure`
- **Indexy k prohledávání:** `logs-api.gw-*`
- **KQL Query (dotaz):**
  ```text
  log.level : "ERROR" AND message : "*Kritická chyba: Nepodařilo se navázat spojení se systémem Centris*"
  ```
- **Frekvence kontroly:** každých 1-5 minut.
- **Akce (Upozornění):** Odeslat na Email/Teams/Slack s textem:
  ```text
  Kritický výpadek spojení! 
  Server: {{context.results.0.host.name}}
  Detail hlášky: {{context.results.0.message}}
  Čas: {{context.date}}
  ```

---

## 2. Import alertu přes JSON (Saved Objects)

V Kibaně se alerty (pravidla) ukládají jako tzv. **Saved Objects**. Ty lze exportovat a importovat ve formátu `.ndjson`. Pro verzi Kibany 8.x se v rámci importu přes Saved Objects používá typ `alert`.

### Postup importu do Kibany:
1. Přejděte do **Management** -> **Stack Management**.
2. V sekci **Kibana** klikněte na **Saved Objects**.
3. Vpravo nahoře klikněte na **Import**.
4. Nahrajte soubor s definicí alertu (např. `centris_connection_alert.ndjson`).
5. **DŮLEŽITÉ:** Po importu klikněte na tři tečky u pravidla a zvolte **Edit**. Následně ho znovu uložte (**Save**). Tím dojde k vygenerování potřebných API klíčů a úloh v Task Manageru.

### Co dělat, pokud importovaný alert nevidíte nebo hlásí chybu:

Pokud import proběhne v pořádku, ale pravidlo v seznamu pravidel (Rules) nevidíte, postupujte následovně:

1.  **Zkontrolujte filtry v seznamu:** V `Stack Management -> Rules` (Pravidla a konektory) zkontrolujte, zda nemáte nastavený filtr na konkrétní typ nebo stav. Zkuste kliknout na **Clear filters** nebo vyhledat název `System Centris` v search baru.
2.  **Ověření v Saved Objects:** Přejděte do `Stack Management -> Saved Objects`. Zadejte do vyhledávání `centris-connection-alert-001`. Pokud zde objekt vidíte, import byl úspěšný a problém je pouze v zobrazení v UI pravidel.
3.  **Špatná sekce (Consumer):** Alerty s `consumer: alerts` se zobrazují v `Stack Management -> Rules`. Pokud byste ho chtěli vidět v sekci `Observability -> Logs -> Alerts`, musel by mít `consumer: logs`. Aktuálně je nastaven jako obecné pravidlo pro indexy.
4.  **Namespace (Space):** Ujistěte se, že jste v Kibaně ve stejném "Space", do kterého jste importovali. Pokud nevíte, zkuste import zopakovat v aktuálním Space.
5.  **Ruční "Save" (Re-save):** Pokud pravidlo v seznamu najdete, ale nefunguje nebo je v "pending" stavu, otevřete ho (**Edit**) a znovu uložte (**Save**). Tím Kibana vygeneruje API klíč a zaregistruje task.
6.  **Verze formátu:** Pro import přes Saved Objects v 8.x používáme v JSONu `"type": "alert"`. Pole `"ruleTypeId"` musí být v tomto formátu vynecháno a nahrazeno `"alertTypeId": ".index-threshold"`, aby import proběhl bez chyby.

## 3. Ukázka definice alertu (NDJSON)

Soubor pro import v Kibaně vypadá následovně (zjednodušený příklad). Celý funkční soubor naleznete v `Elastic/exports/centris_connection_alert.ndjson`.

```json
{
  "id": "centris-connection-alert-001",
  "type": "alert",
  "coreMigrationVersion": "8.12.0",
  "attributes": {
    "name": "System Centris - Connection Failure",
    "consumer": "alerts",
    "alertTypeId": ".index-threshold",
    "schedule": { "interval": "1m" },
    "params": {
      "index": ["logs-api.gw-*"],
      "timeField": "@timestamp",
      "timeWindowSize": 5,
      "timeWindowUnit": "m",
      "thresholdComparator": ">",
      "threshold": [0],
      "kqlQuery": "log.level : \"ERROR\" AND message : \"*Kritická chyba: Nepodařilo se navázat spojení se systémem Centris*\""
    },
    "actions": [],
    "enabled": true,
    "throttle": null,
    "muteAll": false,
    "mutedInstanceIds": [],
    "tags": [],
    "createdAt": "2024-03-26T09:16:00.000Z",
    "updatedAt": "2024-03-26T09:16:00.000Z"
  }
}
```

---

## 4. Výhody JSON/NDJSON exportu
- **Přenositelnost:** Můžete alert vytvořit na DEV prostředí a po otestování ho jednoduchým importem přenést na PROD.
- **Zálohování:** Definice alertů můžete mít uložené v Gitu jako kód (Monitoring as Code).
- **Hromadná správa:** Pomocí skriptů můžete měnit parametry v JSON souborech pro desítky alertů najednou.
