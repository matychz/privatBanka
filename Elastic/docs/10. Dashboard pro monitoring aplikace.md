# 10. Dashboard pro monitoring aplikace

Tento dokument popisuje připravený dashboard pro Elastic (Kibanu), který umožňuje sledovat chybovost, výkon (latenci) a aktivitu operací v rámci systému DigitalnyOnBoarding.

---

## 1. Obsah dashboardu
Dashboard s názvem **Digitalny OnBoarding - Monitoring** obsahuje následující vizualizace:

1.  **Počet chyb v čase**: Vývoj počtu logů s úrovní `ERROR` v čase. Ideální pro rychlou detekci incidentů.
2.  **Úspěšnost operací (bez Health)**: Horizontální "Stacked" sloupcový graf zobrazující rozdělení všech log levelů (`ERROR`, `WARN`, `INFO`, `DEBUG`) pro každou business operaci.
3.  **Top 10 nejdůležitějších operací**: Rozdělení provozu podle typu aktivity.
4.  **Rozdělení log levelů**: Celkový poměr logů napříč systémem.

---

## 2. Prerekvizity
Aby dashboard správně zobrazoval data, musí být splněny následující podmínky:
1.  **Ingest Pipeline**: Musí být aktivní pipeline (viz [dokument č. 4](./4.%20Vytvoření%20Ingest%20Pipeline%20s%20Grok%20procesorem.md)), která extrahuje pole `operation` a `log.level`.
2.  **Správné názvy polí**: Dashboard je připraven pro standardní dynamické mapování v Elasticu. Pro agregace (grafy) se používají pole se sufixem `.keyword` (např. `operation.keyword`, `log.level.keyword`).
3.  **Data View**: Dashboard očekává existenci Data View (Index Patternu) s názvem `logs-api.gw-*` (pokud používáte vlastní indexy) nebo `onboarding-data-view` (který je součástí importu).

---

## 3. Instalace dashboardu
Konfigurace dashboardu je uložena v souboru `Elastic/exports/onboarding_dashboard.ndjson`.

### Postup importu:
1.  V Kibaně přejděte do **Stack Management** -> **Saved Objects**.
2.  Klikněte na tlačítko **Import** v pravém horním rohu.
3.  Vyberte soubor `Elastic/exports/onboarding_dashboard.ndjson` ze svého počítače.
4.  Pokud se objeví dotaz na namapování Index Patternu, vyberte svůj existující Data View pro logy (např. `logs-*`).
    - *Poznámka: NDJSON obsahuje i definici Data View `onboarding-data-view`, takže by se dashboard měl automaticky propojit.*
5.  **DŮLEŽITÉ: Obnova polí v Kibaně**: Pokud po importu vizualizace neukazují data, přejděte do **Stack Management** -> **Data Views**, vyberte svůj index pattern (např. `onboarding-data-view`) a klikněte na ikonu **Reload field list** (vpravo nahoře).
6.  Po dokončení najdete dashboard v sekci **Analytics** -> **Dashboard**.
    - **Tip pro řešení chyb:**
        - Pokud se vizualizace po importu nezobrazí a hlásí chybu `indexPatternLoad requires the "id" argument`, zkuste dashboard importovat znovu se zaškrtnutou volbou **"Create new objects with random IDs"** vypnutou, aby se zachovaly původní reference.
        - Pokud se objeví chyba **„Check your network connection“**, zkuste obnovit stránku (F5) – pravděpodobně vypršela relace přihlášení.

---

## 4. Používaná pole (pro vlastní úpravy)
Pokud budete chtít dashboard dále rozšiřovat, doporučujeme využívat tato pole:
*   `log.level.keyword`: Pro filtraci a rozdělení logů v grafech.
*   `operation.keyword`: Pro seskupování podle typu akce (použito pro agregace).
*   `duration`: Číselné pole pro statistiky rychlosti (vytahováno jako long).
*   `processId` / `correlationId`: Pro proklik do detailu konkrétního průchodu (např. přes tabulku logů).
