# 6. Finální JSON schéma (Reference)

Tento dokument definuje cílovou strukturu logů v systému Elastic pro projekt **DigitalnyOnBoarding**. Všechny komponenty (API Gateway, OfficeLine, Centris) by měly pomocí Ingest Pipelines nebo přímého logování produkovat data v tomto formátu.

---

## Ukázka finálního JSONu

Tento formát je optimalizován pro korelaci napříč distribuovaným systémem a odpovídá požadavkům na audit a debugging.

```json
{
  "timestamp": "2026-03-25T11:00:00.000Z",
  "system": "ibconsole",
  "operation": "ResendOtp",
  "processId": "ca2c6f62f2a946548c4b8e0bd75d4a95",
  "correlationId": "8f2c-5d6e-...",
  "requestId": "92560",
  "sourceSystem": "ibconsole",
  "targetSystem": "office-line",
  "result": "OK",
  "message": "Onboarding.ResendOtp: process ca2c6f62f2a946548c4b8e0bd75d4a95"
}
```

---

## Popis polí

| Pole | Typ | Popis | Příklad |
| :--- | :--- | :--- | :--- |
| **`timestamp`** | Date (ISO8601) | Čas události v UTC. | `2026-03-25T11:00:00.000Z` |
| **`system`** | String | Název komponenty, která log vygenerovala. | `ibconsole`, `centris` |
| **`operation`** | String | Název prováděné akce nebo metody. | `VerifyIdentity`, `Health` |
| **`processId`** | String | Business ID onboardingového procesu. | `ONB-2026-000123` |
| **`correlationId`** | String (GUID) | Technické ID pro sledování celého flow (E2E). | `8f2c-...` |
| **`requestId`** | String | ID konkrétního HTTP požadavku nebo volání. | `c3d4-...` |
| **`sourceSystem`** | String | Systém, který požadavek odeslal. | `mobile`, `office-line` |
| **`targetSystem`** | String | Systém, kterému je požadavek určen. | `centris`, `zenid` |
| **`result`** | String | Výsledek operace (např. OK, Error, Started). | `OK` |
| **`message`** | String | Lidsky čitelný popis události. | `Calling TradingService...` |

---

## Proč toto schéma?

1. **Jednotnost:** Všechny logy vypadají stejně, bez ohledu na to, zda přišly z IIS, Windows Event Logu nebo SQL.
2. **Korelace:** Pomocí `correlationId` uvidíte v Kibaně celou cestu požadavku od mobilu až po Centris.
3. **Audit:** `processId` umožňuje dohledat všechny technické kroky konkrétního klienta pro potřeby bankovního auditu.
4. **Rychlost:** Pole jsou indexovaná, což umožňuje bleskové vyhledávání i v milionech záznamů.
