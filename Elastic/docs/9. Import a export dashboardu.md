# 9. Import a export dashboardu v Kibaně

V Elastic Stacku (Kibaně) je možné snadno přenášet celé dashboardy, vizualizace a další objekty mezi různými prostředími (např. z `dev` do `prod`) pomocí souborů ve formátu **.ndjson**.

---

## 1. Jak exportovat dashboard
Pokud už máte vytvořený dashboard v jednom prostředí a chcete ho přenést jinam:

1.  V Kibaně přejděte do **Stack Management** -> **Saved Objects**.
2.  Do vyhledávacího pole zadejte název svého dashboardu.
3.  Zaškrtněte políčko u dashboardu.
4.  Klikněte na tlačítko **Export**.
5.  V dialogu ponechte zapnutou volbu **Include related objects** (to zajistí, že se vyexportují i všechny vizualizace a mapy, které dashboard používá).
6.  Stáhněte si vygenerovaný soubor (např. `export.ndjson`).

---

## 2. Jak importovat dashboard
Pro nahrání dashboardu do cílového prostředí:

1.  Otevřete cílovou Kibanu.
2.  Přejděte do **Stack Management** -> **Saved Objects**.
3.  V pravém horním rohu klikněte na **Import**.
4.  Vyberte svůj `.ndjson` soubor.
5.  Pokud se objeví konflikt (např. stejné ID objektu), můžete zvolit **Overwrite** (přepsat) nebo **Check for existing objects**.
6.  Klikněte na **Import**.

---

## 3. Možné problémy (Troubleshooting)

### Chyba: „Check your network connection and try again.“
Tato chyba se v Kibaně objevuje nejčastěji při importu Saved Objects nebo při práci v Dev Tools.
*   **Příčina 1: Vypršení relace (Session Timeout).** Vaše přihlášení mohlo vypršet.
    *   **Řešení:** Zkuste obnovit stránku prohlížeče (F5) a případně se znovu přihlásit.
*   **Příčina 2: Velikost požadavku.** I když je soubor `.ndjson` malý, síťové prvky (proxy, firewall) mohou blokovat určité typy POST požadavků.
    *   **Řešení:** Zkuste soubor importovat znovu. Pokud používáte VPN, zkuste ji dočasně vypnout/zapnout pro ověření stability spojení.
*   **Příčina 3: Dočasný výpadek Elastic Cloud / Serverless.**
    *   **Řešení:** Počkejte několik minut a zkuste to znovu. Zkontrolujte stav své instance v Elastic Cloud Console.
*   **Příčina 4: Blokování prohlížečem (Ad-block / Firewall).** Některá rozšíření prohlížeče mohou blokovat síťové požadavky Kibany.
    *   **Řešení:** Zkuste import v anonymním okně prohlížeče (Incognito mode).

### Chybějící Data View (Index Pattern)
Při importu se může stát, že Kibana nahlásí chybu: *"The following objects could not be imported because their references are missing: [Data View ID]"*.
*   **Příčina:** Dashboard odkazuje na Data View (např. `logs-*`), který v novém prostředí neexistuje pod stejným ID.
*   **Řešení:** V průběhu importu vám Kibana nabídne možnost namapovat tyto reference na existující Data View v novém prostředí. Vyberte odpovídající Data View ze seznamu.

### Verze Kibany
Doporučuje se exportovat a importovat mezi stejnými nebo blízkými verzemi Elasticu. Import ze starší verze do novější (např. 8.x do 9.x) obvykle funguje bez problémů, opačně to může být problematické.

## 4. Tip: Automatizace přes API
Pokud potřebujete importovat dashboardy automatizovaně (např. v rámci CI/CD), můžete použít Kibana Saved Objects API:

```bash
curl -X POST "https://your-kibana-url:5601/api/saved_objects/_import?overwrite=true" \
  -H "kbn-xsrf: true" \
  --form file=@export.ndjson \
  -u username:password
```
