# 11. Anonymizace osobních údajů (GDPR)

Tento dokument popisuje, jak v rámci Ingest Pipeline anonymizovat citlivé údaje, jako jsou e-mailové adresy, aby byly v souladu s pravidly GDPR, ale zároveň zůstala zachována možnost korelity (stejný e-mail má vždy stejný hash).

---

## 1. Požadovaný formát
E-mailová adresa (např. `jan.novak@example.com`) bude v logu nahrazena maskovaným tvarem:
`#EMAIL#<md5_hash>#EMAIL#`

**Příklad:**
- Původní zpráva: `Odeslán e-mail na adresu user123@example.com`
- Anonymizovaná zpráva: `Odeslán e-mail na adresu #EMAIL#757d2a9f4c3c3a05953507c6f370a41d#EMAIL#`

---

## 2. Implementace pomocí MD5 (Grok + Fingerprint + Script)

Pro dosažení skutečného **MD5** hashe (namísto prostého `hashCode()`) využijeme v Ingest Pipeline kombinaci tří procesorů. Tento postup zajistí, že e-mail bude extrahován, zahashován standardním algoritmem MD5 a následně bezpečně nahrazen v původní zprávě.

### Kód procesorů pro Ingest Pipeline:

Vložte tyto procesory do vaší pipeline (např. před finální vyčištění zprávy):

```json
[
  {
    "grok": {
      "description": "A. Extrakce e-mailu do dočasného pole",
      "field": "message",
      "patterns": ["%{EMAILADDRESS:tmp_email_raw}"],
      "ignore_failure": true
    }
  },
  {
    "set": {
      "description": "B. Příprava e-mailu pro hashování (kopie)",
      "field": "tmp_email_norm",
      "copy_from": "tmp_email_raw",
      "ignore_empty_value": true
    }
  },
  {
    "lowercase": {
      "description": "C. Normalizace e-mailu na malá písmena (pro korelaci)",
      "field": "tmp_email_norm",
      "ignore_missing": true
    }
  },
  {
    "fingerprint": {
      "description": "D. Výpočet MD5 hashe",
      "fields": ["tmp_email_norm"],
      "target_field": "tmp_email_hash",
      "method": "MD5",
      "ignore_missing": true
    }
  },
  {
    "script": {
      "description": "E. Nahrazení e-mailu v poli message hashem",
      "if": "ctx.tmp_email_raw != null && ctx.tmp_email_hash != null",
      "source": "ctx.message = ctx.message.replace(ctx.tmp_email_raw, '#EMAIL#' + ctx.tmp_email_hash + '#EMAIL#')"
    }
  },
  {
    "remove": {
      "description": "F. Úklid dočasných polí",
      "field": [
        "tmp_email_raw",
        "tmp_email_norm",
        "tmp_email_hash"
      ],
      "ignore_missing": true
    }
  }
]
```

---

## 3. Kam procesor zařadit?

Doporučujeme tento procesor zařadit do vaší existující pipeline (`api-gw-pipeline`) **před** finální vyčištění zprávy, nebo jako jeden z prvních kroků po úvodním Groku.

### Výhody tohoto přístupu:
1.  **Bezpečnost:** Skutečné e-maily se nikdy neuloží do indexu (pokud anonymizujete pole `message`).
2.  **Korelace:** Pokud uživatel nahlásí problém, můžete si vypočítat MD5 jeho e-mailu a vyhledat v Elasticu všechny logy, které se ho týkají, aniž byste e-mail znali.
3.  **Výkon:** Regulární výraz je cílený pouze na pole `message` a spouští se pouze v Ingest Pipeline (při zápisu).
